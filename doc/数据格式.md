# 数据格式

1. **文件路径**：/bridges/protocol/ros

2. **消息格式**
    
    - 控制指令：protocol::msg::MotionServoCmd
    
        数据发布参考**键盘控制程序**实现
        
        ```
        int32        motion_id
        int32        cmd_type
        int32        cmd_source   
        int32        value
        float32[]    vel_des      # x y yaw speed  m/s
        float32[]    rpy_des      # roll pitch yaw  rad
        float32[]    pos_des      # x y z  m/s
        float32[]    acc_des      # acc for jump m^2/s
        float32[]    ctrl_point   # pose ctrl point  m
        float32[]    foot_pose    # front/back foot pose x,y,z  m
        float32[]    step_height  # step height when trot  m

        # cmd_type 取值定义
        int32 SERVO_START = 0
        int32 SERVO_DATA = 1
        int32 SERVO_END = 2

        # cmd_source 取值定义
        int32 APP = 0
        int32 AUDIO = 1
        int32 VIS = 2
        ```

        **motion_id定义：**/bridges/protocol/ros/msg/MotionID.msg
        
        motion_id仅需使用以下内容：

        ```
        int32 ESTOP = 0                 # 急停
        int32 GETDOWN = 101             # 趴下
        int32 RECOVERYSTAND = 111       # 站立
        int32 WALK_USERTROT = 303       # 慢走模式
        
        ```
        以及
        ```
        motion_id = 308                 # 快走模式
        位于路径：/motion/motion_action/preset/motion_id_map.toml
        ```

        **cmd_type字段内容：**
        
        ```
        int32 SERVO_START = 0
        int32 SERVO_DATA = 1
        int32 SERVO_END = 2
        ```

        使用时代码中填充

        ```
        protocol::msg::MotionServoCmd::SERVO_START
        protocol::msg::MotionServoCmd::SERVO_DATA
        protocol::msg::MotionServoCmd::SERVO_END
        ```

        cmd_source字段可使用protocol::msg::MotionServoCmd::APP

        value填2即可

        step_height固定为0.05即可

        由于场地为平地，所以使用vel_des，发送x y yaw速度即可，具体数值可根据**路径规划**部分计算的速度数据。

    - 里程计数据：nav_msgs/msg/Odometry

        主要使用pose和orientation字段的数据，pose字段包括x,y,z位置信息，orientation字段为姿态四元数，需进行转换为欧拉角表示机器人朝向
