1. 模型文件
检查以下内容：
```bash
ls -l /SDCARD/miloc/models
cat /SDCARD/miloc/models/version.toml

```

2. 重定位地图
检查默认地图map_1
```bash
find /SDCARD/miloc -maxdepth 4 | grep map_1
```

3. 启动Miloc——自启动时发起的miloc_server不可用时自行启动

进入cyberdog2的板子，namespace根据实际情况决定

**miloc_server**
```bash
ros2 run cyberdog_miloc miloc_server --ros-args   -r __ns:=/cyberdog_1   -r __node:=miloc_server   -p config_path:=/opt/ros2/cyberdog/share/cyberdog_miloc/config/config.yml   -p cam0_topic:=image_rgb   -p cam1_topic:=image_left   -p cam2_topic:=image_right   -p odom_slam:=odom_slam   -p odom_out:=odom_out   -p reloc_failure_threshold:=20   -p immediately_reconstruct:=true
```

**占有图启动**
```bash
ros2 launch navigation_bringup node.occmap.launch.py namespace:=/cyberdog_1
```

4. miloc_status状态

节点启动，未建图

```bash
miloc_status: 1
current_map_id: -1
```

5. 启动 Mivins Mapping

```bash
ros2 launch mivins dog_d430i_stereo_odometry_mapping.py namespace:=/cyberdog_1
```

节点生命周期激活

```bash
ros2 lifecycle set /cyberdog_1/mivinsmapping configure
ros2 lifecycle set /cyberdog_1/mivinsmapping activate
```

此时miloc_status为6

6. 启动建图服务

```bash
ros2 service call /cyberdog_1/start_vins_mapping std_srvs/srv/SetBool "{data: true}"
```

7. 采集建图数据
让机器人在目标区域内移动，保证：

有足够回环和覆盖
不是只原地转圈
不是直线短距离来回
图像清晰，不要太快
尽量经过关键转角、走廊、开阔区


建议最少做到：

起点附近停留几秒
缓慢走一段
转弯
回到附近形成一定闭环
覆盖主要定位区域

8. 结束建图

```bash
ros2 service call /cyberdog_1/stop_vins_mapping cyberdog_visions_interfaces/srv/FinishMap "{finish: true, map_name: 'map_1'}"
```

9. 等待重建完成

先观察 Miloc 状态：

```bash
ros2 topic echo /cyberdog_1/miloc_status
```
重建期间通常会进入 RECONSTRUCTING，重建完成后回到 INACTIVE。

再轮询地图状态：

```bash
ros2 service call /cyberdog_1/get_miloc_status cyberdog_visions_interfaces/srv/MilocMapHandler "{map_id: 1}"
```

返回结果：

```bash
code=0
message='reloc map ok'
```
**错误状态：**

300：还在 mapping / reconstructing 过程中
301：未完成
302：地图错误

10. 检查map_1文件

```bash
find /SDCARD/miloc/maps/map_1 -maxdepth 3 -type f
```
